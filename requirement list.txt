1번
requirement : 회원은 자전거를 반납할 수 있어야 한다
use case : 자전거 반납

2번
requirement : 시스템은 회원이 자전거 반납 시 사용 시간을 기준으로 요금을 계산하여 결제를 시도해야한다
use case : 자전거 반납

3번
requirement : 회원은 요금 조회 화면에서 대여 시간 및 요금을 볼 수 있어야 한다
use case : 요금 조회

4번
requirement : 회원은 과거 대여 기록을 조회할 수 있어야 한다
use case : 과거 대여 기록 조회

5번
requirement : 회원은 과거 대여 기록을 조회할 때 정렬 기준을 날짜 또는 대여소로 선택할 수 있어야 한다
use case : 과거 대여 기록 정렬 기준 변경

6번
requirement : 회원은 과거 대여 기록 중 원하는 항목을 삭제할 수 있어야 한다
use case : 과거 기록 삭제

7번
requirement : 관리자는 자전거 대여 정보를 반납 시간 기준 내림차순으로 조회할 수 있어야 한다
use case : 자전거 대여 통계 조회

8번
requirement : 관리자는 자전거 대여 정보를 조회할 때 정렬 기준을 반납 시간 또는 지역으로 선택할 수 있어야 한다
use case : 자전거 대여 통계 정렬 기준 변경

9번
requirement : 관리자는 자전거 대여 정보를 최근 1주일, 1개월, 1년 단위로 총 대여 금액 및 대여 횟수를 조회할 수 있어야 한다
use case : 일/연/월 단위 대여 금액 및 횟수 소계 조회
