1번
requirement : 회원이 현재 대여 중인 자전거를 조회할 수 있어야 한다.
use case : 자전거 대여 정보 조회

2번
requirement : 회원이 자전거 예약대기 정보를 조회할 수 있어야 한다. 또한, 원할 경우 예약 대기를 취소할 수 있어야 한다.
use case : 자전거 예약대기 정보 조회

3번
requirement : 회원이 특정 자전거를 지정된 대여소에 반납할 수 있어야 한다. 자전거 반납 후에는 원할 경우 위치 정보를 기반으로 근처 식당을 추천받아서 예약할 수 있는 외부 서비스와 연결될 수 있어야 한다.
반납된 자전거의 대기 1순위 회원에게는 예약되었다는 이메일을 보낸다.
use case : 자전거 반납

4번
requirement : 회원은 회원가입을 할 수 있어야 한다. ID, 비밀번호, 전화번호, 결제 수단, 선호 자전거 유형등의 필수 입력 정보를 입력해야한다.
use case : 회원가입

5번
requirement : 회원은 회원탈퇴를 할 수 있어야 한다. 탈퇴 시 모든 이용 권한과 데이터가 삭제된다.
use case : 회원탈퇴

6번
requirement : 유저는 로그인을 할 수 있어야 한다. 관리자는 주어진 ID와 비밀번호로 로그인한다.
use case : 로그인

7번
requirement : 유저는 로그아웃을 할 수 있어야 한다. 로그아웃 시 시스템 접속이 종료된다.
use case : 로그아웃

8번
requirement : 관리자는 대여소 정보를 등록할 수 있어야 한다. 등록 시 대여소 이름, 위치 등을 기재해야한다.
use case : 대여소 정보 등록

9번
requiremet : 관리자는 대여소 리스트를 조회할 수 있어야 한다. 관리자는 특정 대여소 항목을 삭제할 수 있다.
use case : 대여소 리스트 조회

10번
requirement : 관리자는 대여소 상세내용을 조회할 수 있어야 한다.
use case : 대여소 상세내용 조회

11번
requirement : 회원은 자전거를 반납할 수 있어야 하며 시스템은 회원이 자전거 반납 시 사용 시간을 기준으로 요금을 계산하여 결제를 시도해야한다.
use case : 자전거 반납

12번
requirement : 회원은 요금 조회 화면에서 반납한 대여에 대해서 대여 시간 및 요금을 볼 수 있어야 한다.
use case : 요금 조회

13번
requirement : 회원은 과거 대여 기록을 조회하고 원하면 삭제할 수 있어야 한다. 기록들은 기본적으로 날짜 기준으로 정렬되며 회원은 정렬 기준을 대여소로 변경할 수 있어야한다.
use case : 과거 대여 기록 조회

14번
requirement : 관리자는 대여소 이름, 대여소 위치, 자전거 ID, 자전거 제품명, 자전거 유형으로 구성된 자전거 대여 정보를 조회할 수 있어야 한다. 정보들은 기본적으로 시간 기준 내림차순으로 정렬되며 관리자는 정렬 기준을 도시와 주소의 오름차순으로 변경할 수 있어야 한다.
use case : 자전거 대여 통계 조회

15번
requirement : 관리자는 자전거 대여 정보를 최근 1주일, 1개월, 1년 단위로 총 대여 금액 및 대여 횟수를 조회할 수 있어야 하며 기간을 1주일, 1개월, 1년 중 하나로 변경할 수 있어야 한다.
use case : 주/월/연 단위 대여 금액 및 횟수 소계 조회
