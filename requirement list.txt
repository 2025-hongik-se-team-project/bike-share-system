1번
requirement : 회원이 현재 대여 중인 자전거를 조회할 수 있어야 한다
use case : 자전거 대여 정보 조회

2번
requirement : 회원이 자전거 예약대기 정보를 조회할 수 있어야 한다
use case : 자전거 예약대기 정보 조회

3번
requirement : 예약 대기는 취소할 수 있어야 한다
use case : 자전거 예약대기 취소

4번
requirement : 회원이 특정 자전거를 지정된 대여소에 반납할 수 있어야 한다
use case : 자전거 반납

5번
requirement : 회원은 자전거 반납 후에 원할 경우 위치 정보를 기반으로 근처 식당을 추천받아서 예약할 수 있는 외부 서비스와 연결될 수 있어야 한다
use case : 외부 식당 예약 서비스 연결

6번
requirement : 관리자는 자전거 ID, 자전거 제품명, 유형(일반/전기), 소속 대여소, 상태(사용 가능/수리 중) 등의 자전거 정보를 등록한다.
use case : 자전거 정보 등록

7번
requirement : 관리자는 등록된 자전거 리스트를 조회한다. 관리자는 원하는 자전거의 상세내용을 확인하거나 특정 자전거 항목을 삭제한다.
use case : 자전거 리스트 조회

8번
requirement : 회원은 이름을 통해 조건에 맞는 대여소를 검색한다.
use case : 대여소 검색

9번
requirement : 회원은 대여소 이름, 대여소 위치, 사용 가능 자전거 목록 등의 상세정보를 조회한다. 회원은 자전거를 즉시 대여하거나 예약대기를 신청한다.
use case : 대여소 상세정보 조회
